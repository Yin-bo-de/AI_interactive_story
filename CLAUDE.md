## 项目概述

**幻境回响** — 互动式AI推理叙事游戏（PC端网页）。用户通过与AI角色（如探长）互动，共同推进剧情，在30分钟内体验一个完整的推理故事。

## 技术架构

### 后端架构
- **框架**: Python FastAPI
- **核心服务**:
  - 会话管理：处理用户会话状态
  - 上下文存储：维护完整的对话历史和关键信息
  - LLM API交互：与AI模型通信
  - 计时与轮次控制：30分钟倒计时 + 100轮上限
- **部署**: 云服务器

### 前端架构
- **框架**: React 或 Vue.js 单页应用
- **部署**: Vercel/Netlify 或自建服务器
- **UI特点**: PC端宽屏适配，沉浸式阅读体验

### 数据存储
- 初期使用内存缓存或简单文件存储
- 后续可扩展为数据库（用户历史、数据统计）

### AI模型集成
- 接入顶级推理/对话大模型（OpenAI GPT-4、Anthropic Claude、智谱AI等）
- 使用系统提示 + 上下文的方式调用API

## 核心机制

### 秘密结局种子机制
1. **开局**: AI生成故事背景的同时，在内部生成3个可能的结局种子（如：凶手是管家/商业伙伴/意外）
2. **推进**: 根据用户输入，选择最合适的种子靠拢，逐步丰满线索
3. **结局**: 根据实际选择的路径生成最终结局

### 上下文管理策略
- **全量存储**: 后端维护完整对话历史
- **滑动窗口**: 发送给API时采用"关键摘要 + 最近N轮对话"
- **关键信息**: 故事背景、人物、线索、结局种子作为固定系统提示
- **状态结构化**: 维护JSON格式的剧情状态

### 游戏控制机制
- **30分钟倒计时**: 后端启动，独立于前端
- **100轮上限**: 每完成一次"用户输入->AI回复"循环，计数器+1
- **结束条件**: 时间或轮次任一先到即触发结束

## AI Prompt 设计

### 开局 Prompt
- 初始化故事背景和人物
- 秘密生成3个结局种子
- 设置AI角色（如探长·林默）
- 引导用户开始互动

### 互动推进 Prompt
- 角色扮演：以角色身份回复
- 剧情推进：根据输入靠拢结局种子
- 互动引导：提供1-3个下步行动建议
- 温和纠正：偏离主线时拉回
- 创作配合：采纳合理的用户设定

### 游戏结束 Prompt
- 根据选定的结局种子生成完整结局
- 逻辑自洽，呼应关键线索
- 回顾用户贡献

## 开发规范

### Python 环境
- 使用虚拟环境：`venv` 文件夹
- 激活环境：`source venv/bin/activate`

### 日志规范
- 使用 loguru 插件记录日志
- 日志级别：ERROR、WARNING、INFO
- 日志目录：`log/`，以日期命名（如 `2023-03-15.log`）

### 代码质量
- 遵循 Python 最佳实践（PEP 8）
- 添加必要的中文注释
- 类型提示（Type Hints）
