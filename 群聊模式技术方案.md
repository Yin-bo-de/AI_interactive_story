# 群聊模式技术方案

## Context
当前"幻境回响"游戏采用用户与单个AI角色一对一聊天的模式推进剧情。现需要改造为**群聊模式**：将所有剧本人物加入同一个聊天群，用户与所有AI角色在群内共同交流，AI角色之间可以互相@提及对话，自动在适当的时机发言，共同分享线索、推进剧情演进。

## 整体设计思路
基于现有FastAPI+React技术栈，最小化改动，复用现有会话管理、结局种子、计时轮次控制等核心逻辑，新增多角色管理和群聊调度机制。

---

## 一、后端架构改造

### 1. 数据结构扩展
#### 会话状态新增字段：
```python
class SessionState(BaseModel):
    # 原有字段保留
    session_id: str
    story_background: str
    ending_seeds: List[EndingSeed]
    current_ending: str
    dialogue_history: List[Message]
    round_count: int
    start_time: datetime
    is_finished: bool

    # 新增字段
    characters: List[Character]  # 所有参与群聊的角色列表
    current_speaker_queue: List[str]  # 待发言角色队列
    last_speaker: Optional[str]  # 上一个发言的角色
```

#### 角色数据结构：
```python
class Character(BaseModel):
    id: str  # 角色唯一标识
    name: str  # 角色名称
    avatar: str  # 头像URL
    description: str  # 角色背景设定
    personality: str  # 性格特点
    speaking_style: str  # 说话风格
    priority: int  # 角色优先级，数字越小优先级越高（1=主控角色，2=重要角色，3=普通角色）
    is_active: bool  # 是否处于活跃状态（是否已加入群聊并可以主动发言）
    last_speak_time: Optional[datetime]  # 上次发言时间
    known_clues: List[str]  # 该角色掌握的线索列表
    relationships: Dict[str, str]  # 与其他角色的关系，key为角色ID，value为关系描述
```

#### 消息结构扩展：
```python
class Message(BaseModel):
    id: str
    sender_id: str  # 发送者ID（user/角色ID）
    sender_name: str  # 发送者名称
    sender_avatar: Optional[str]  # 发送者头像
    content: str
    timestamp: datetime
    mentioned_characters: List[str]  # @提及的角色ID列表
    message_type: str  # text/action/system
```

### 2. 核心逻辑改造
#### （1）群聊消息调度器
新增`GroupChatScheduler`服务，负责调度AI角色发言：
- **触发时机**：
  - 用户发送消息后，触发1-3个相关角色发言
  - 某个AI角色发言后，判断是否需要触发其他被@提及的角色回复
  - 空闲超过30秒时，触发关键角色发言推进剧情
  - 剧情节点触发时，解锁新角色加入群聊
- **调度策略**：
  - 被@提及的角色优先回复
  - 持有相关线索的角色优先发言
  - 角色优先级高的角色优先发言（如探长等主控角色）
  - 避免同一角色连续发言
  - 每轮用户消息后，AI角色总发言数控制在2-5条
  - 多个角色同时生成回复时，随机排序逐个输出
- **发言冷却**：每个角色发言后有10-30秒冷却时间，避免刷屏
- **角色出场管理**：支持角色逐步解锁加入群聊，根据剧情进展动态添加新角色

#### （2）多角色AI调用逻辑
改造原有LLM调用服务，支持多角色并行调用：
- 每个角色使用独立的prompt模板，包含角色专属设定
- 输入上下文包含：群聊历史+角色专属设定+当前剧情状态
- 支持批量调用多个角色的回复，提升响应速度
- 自动判断回复相关性，过滤不符合角色设定的发言

#### （3）@提及解析
新增提及解析功能：
- 识别消息中@的角色名称，自动关联到角色ID
- 被@的角色优先获得回复权
- 支持在回复中自动@相关角色

### 3. API接口调整
#### 新增/修改接口：
| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/chat/send` | POST | 用户发送消息，返回待发言的角色列表 |
| `/api/chat/characters` | GET | 获取当前会话所有角色列表 |
| `/api/chat/next-speech` | GET | 轮询获取下一个AI角色的发言 |
| `/api/chat/history` | GET | 获取完整群聊历史记录 |

#### SSE实时推送（可选优化）：
新增SSE接口`/api/chat/stream`，实时推送AI角色的发言，避免前端轮询。

## 三、Prompt体系改造

### 1. 角色设定Prompt
每个角色独立的设定模板：
```
你现在扮演的是剧本中的【{角色名称}】。
角色背景：{角色背景描述}
性格特点：{性格描述}
说话风格：{说话风格，比如：冷静严谨/活泼跳脱/语速缓慢/喜欢用歇后语等}
你掌握的线索：{该角色已知的线索列表}
与其他角色的关系：{你和其他角色的关系字典，比如：{"guan_jia": "王管家是你的上司，你怀疑他偷了东西"}}

群聊规则：
1. 你在群聊中发言，符合你的身份和性格
2. 有人@你时，必须回应
3. 讨论到你掌握的线索时，你可以主动分享
4. 不要重复其他人已经说过的内容
5. 回答要简洁自然，符合聊天场景，不要太长
6. 你可以@其他角色，和他们对话
7. 如果你不知道怎么回答，可以说"我不太清楚这件事"或者保持沉默
```

### 2. 群聊调度Prompt
判断哪些角色需要发言的调度模板：
```
当前群聊上下文：
{最近10条消息}

当前剧情状态：
{当前剧情进展}

可用角色列表：
{所有角色及其设定}

请根据上下文判断，接下来应该有哪些角色发言？按照优先级排序，最多选3个角色，并说明每个角色应该回复的核心内容。
返回格式：
[
  {"character_id": "角色ID", "core_content": "回复的核心内容"},
  ...
]
```

### 3. 结局触发逻辑
原有结局种子机制保留，调整为：
- 当剧情推进到关键节点时，由主控角色（如探长）发起总结
- 其他角色可以补充相关线索
- 最终结局由系统根据所有线索和对话历史生成

## 四、核心机制兼容调整

### 1. 计时与轮次控制
- 30分钟倒计时逻辑保留
- 轮次计数调整为：用户发送1条消息 + 所有AI回复 = 1轮
- 100轮上限逻辑不变

### 2. 结局种子机制
- 开局生成3个结局种子逻辑保留
- 所有角色的发言都会被纳入剧情走向判断
- 系统根据整体对话内容自动选择最合适的结局方向

### 3. 上下文管理
- 全量存储所有群聊历史
- 发送给每个角色的上下文采用：固定系统提示 + 角色专属设定 + 最近20条群聊消息
- 关键剧情信息全局共享

## 六、验证方案
1. **功能测试**：
   - 开局是否正确创建初始角色列表，未解锁角色不会出现在群聊中
   - 剧情推进到特定节点时，相关角色是否自动解锁加入群聊
   - 用户发送消息后，是否有相关的AI角色回复
   - @某个角色时，该角色是否会优先回复
   - AI角色之间是否可以互相@对话
   - 所有角色的发言是否符合其设定
   - 高优先级角色是否会优先发言引导剧情
   - 多个角色回复时，是否按顺序逐个输出，不会同时刷屏
2. **剧情推进测试**：
   - 线索是否能通过不同角色逐步披露
   - 剧情是否能正常推进到结局
   - 结局是否符合逻辑，呼应之前的线索
3. **性能测试**：
   - 同时有3-5个角色发言时，响应速度是否符合要求
   - 消息不会丢失或重复
   - 100轮对话后系统运行稳定

## 七、优势分析
1. **复用现有架构**：大部分核心逻辑（会话管理、结局种子、计时等）都可以复用，改动量可控
2. **体验提升**：多角色群聊更有沉浸感，剧情推进更自然
3. **可扩展性**：支持任意数量的角色参与，适合复杂剧本
4. **兼容性**：可以通过配置开关支持单角色/群聊两种模式，兼容旧剧本

---

## 八、NOT in scope（不在本次方案范围内）
1. **语音聊天功能**：本次方案仅支持文本消息，不涉及语音输入输出
2. **实时音视频通话**：不支持视频/语音通话功能
3. **多用户联机**：本次方案仅支持单用户与多AI角色群聊，不支持多用户同时在线互动
4. **表情/图片/文件发送**：仅支持纯文本消息和@提及功能，不支持富媒体消息
5. **角色AI记忆个性化**：所有角色共享全局剧情状态，不做独立的角色长期记忆差异化
6. **移动端适配**：本次方案聚焦PC端体验，移动端适配后续迭代考虑
